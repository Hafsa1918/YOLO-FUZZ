# Contributing to YOLO-FUZZ

First off, thank you for considering contributing to this project! It’s people like you who make the open-source community such a fantastic place to learn and build.

We want to make contributing as easy and transparent as possible, whether it's:

- Reporting a bug
- Discussing the current state of the code
- Submitting a fix
- Proposing new features

## 📜 Code of Conduct
By participating in this project, you agree to maintain a professional and respectful environment for all contributors.

## Reporting Issues/Bugs

If you find a bug or have a suggestion, please open an issue:

1. Go to the **Issues** tab at the top of the repository.
   
2. Click the **New Issue** button.
   
3. Use a descriptive title and provide as much detail as possible (steps to reproduce, screenshots, etc.).

## Pull Requests

We welcome pull requests! To contribute:
1. **Fork** the Project.
2. **Create** your Feature Branch.
3. **Commit** your Changes.
4. **Push** to the Branch.
5. **Open** a Pull Request.

*Note: Please ensure your Pull Request description clearly explains the changes made.*

### Branch Naming

Please follow this naming convention for branches:

- `feature/feature-name` (for new features)
- `fix/bug-name` (for bug fixes)
- `docs/update-name` (for documentation)

### Coding Style
- We follow the [Standard Style](https://peps.python.org/pep-0008/) (PEP8 for Python).
- Ensure your code is properly commented.
- Run tests before submitting a Pull Request.
